## Highlights


### New option `checkRelationalComparisons` in `no-constant-binary-expression`

ESLint v10.6.0 introduces a new option `checkRelationalComparisons` for the [`no-constant-binary-expression`](/docs/head/rules/no-constant-binary-expression) rule.
When enabled, the rule reports relational comparisons using `<`, `<=`, `>`, or `>=` whose result is always constant based on their literal operands.

For example:

```js
const value = "a" > "b"; // always `false`
```

```js
while (0 <= 0) { // always `true`
    /* ... */
}
```

### Rule refinements

The following rules have been tweaked to improve correctness and ensure more consistent or intuitive behavior in edge cases:

* [`max-classes-per-file`](/docs/latest/rules/max-classes-per-file)
* [`max-nested-callbacks`](/docs/latest/rules/max-nested-callbacks)
* [`no-constant-binary-expression`](/docs/latest/rules/no-constant-binary-expression)
* [`no-extra-boolean-cast`](/docs/latest/rules/no-extra-boolean-cast)
* [`no-promise-executor-return`](/docs/latest/rules/no-promise-executor-return)
* [`no-throw-literal`](/docs/latest/rules/no-throw-literal)
* [`prefer-exponentiation-operator`](/docs/latest/rules/prefer-exponentiation-operator)
* [`prefer-promise-reject-errors`](/docs/latest/rules/prefer-promise-reject-errors)
* [`radix`](/docs/latest/rules/radix)
