## Highlights

* Five core rules now highlight smaller ranges of code to avoid shadowing other problems in editors.
  * Rules [`max-lines-per-function`](/docs/latest/rules/max-lines-per-function), [`max-nested-callbacks`](/docs/latest/rules/max-nested-callbacks), and [`max-statements`](/docs/latest/rules/max-statements) now highlight only the function header instead of the entire function.
  * Rules [`max-depth`](/docs/latest/rules/max-depth) and [`no-with`](/docs/latest/rules/no-with) now highlight only the first keyword.
* Several errors in the calculations have been corrected in the [`max-depth`](/docs/latest/rules/max-depth) and [`max-nested-callbacks`](/docs/latest/rules/max-nested-callbacks) rules. These bug fixes can result in reporting more linting errors in existing code.
